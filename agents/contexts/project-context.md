# Project Jun Fan — Project Context

## Philosophy

**"Absorb. Refine. Build."**

- **Absorb** — Study what works in martial arts, design, and engineering. Take inspiration without copying.
- **Refine** — Strip away the unnecessary. Improve clarity, performance, and intention.
- **Build** — Create something purposeful, beautiful, and maintainable.

Jun Fan is Bruce Lee's Chinese given name. This project embodies his martial arts philosophy of simplicity, directness, and freedom — applied to software.

## What Project Jun Fan Is

- A modern, front-end first web application built with Next.js 16 and React 19
- A design system (Horizon) that enforces visual and interaction consistency
- A zero-backend architecture — all content is static JSON
- A portfolio of products (apps, tools, showcases) built on shared infrastructure
- A playground for exploring cutting-edge web technologies
- An opinionated monorepo with clear conventions for features, components, and content

## What Project Jun Fan Is Not

- It is not a backend service — there is no database, API server, or authentication layer
- It is not a CMS — content is hand-authored JSON, not managed through an admin panel
- It is not a starter kit or boilerplate — it is a specific project with specific goals
- It is not a library or framework — it is an application built *with* libraries and frameworks

## Guiding Principles

1. **Simplicity over complexity** — Every abstraction must earn its place.
2. **Consistency over convenience** — Follow the design system even when shortcuts are tempting.
3. **Performance is a feature** — Lighthouse scores, bundle size, and runtime perf matter.
4. **Content is code** — JSON files are version-controlled, reviewed, and tested like source code.
5. **Design is engineering** — Design tokens live in code; designers and engineers share the same source of truth.
6. **Move deliberately** — Small, focused PRs with clear scope and thorough reviews.
