# Product Manager — Validation Checklist

## Product Vision & Strategy
- [ ] Product vision is documented and up to date
- [ ] Current quarter goals are defined and measurable
- [ ] Stakeholders have signed off on priorities
- [ ] Zero-backend constraint is respected in all plans

## Backlog Health
- [ ] Top 10 items have clear acceptance criteria
- [ ] Every story follows the problem/solution/success format
- [ ] No item exceeds 2 sprints of work without being split
- [ ] Dependencies between stories are documented
- [ ] Out-of-scope items are explicitly listed

## User Stories
- [ ] Story has a clear user persona ("As a...")
- [ ] Acceptance criteria are testable (pass/fail)
- [ ] Edge cases are considered (empty state, error state, loading state)
- [ ] JSON-driven content strategy is defined
- [ ] Story is independent, negotiable, valuable, estimable, small, testable (INVEST)

## Prioritization
- [ ] Each item has an effort estimate (S/M/L/XL)
- [ ] Each item has an impact rating (High/Medium/Low)
- [ ] Prioritization follows effort vs. impact matrix
- [ ] Technical debt items are represented in the backlog

## Review Readiness
- [ ] Completed work matches acceptance criteria
- [ ] Demo scenario is prepared
- [ ] Stakeholder feedback is documented
- [ ] Go/no-go decision is recorded
