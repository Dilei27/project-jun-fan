# Roadmap Planner — Validation Checklist

## Structure
- [ ] Quarter and year are specified
- [ ] 2-3 themes defined per quarter
- [ ] Each theme has 2-4 initiatives
- [ ] Every initiative has a named DRI
- [ ] Every initiative has an effort estimate (S/M/L/XL)

## Capacity
- [ ] Total allocated does not exceed 80% of capacity
- [ ] Buffer is explicitly reserved (20%)
- [ ] Capacity model includes design, engineering, and content

## Dependencies
- [ ] All external dependencies are documented per initiative
- [ ] Critical path is identified
- [ ] Dependency owners have confirmed availability
- [ ] No initiative is blocked by an unconfirmed dependency

## Risk Management
- [ ] Top 3 risks are documented with mitigations
- [ ] Each risk has a probability (Low/Med/High) and impact (Low/Med/High)
- [ ] Mitigation owner is assigned

## Communication
- [ ] Stakeholders have been informed of the current roadmap
- [ ] Changes from the previous roadmap are highlighted
- [ ] Decision deadlines are clear
- [ ] Roadmap is accessible in a shared location (e.g., Notion, Google Docs)

## Alignment
- [ ] Roadmap aligns with product vision and goals
- [ ] Zero-backend constraint is respected
- [ ] Design system dependencies are accounted for
- [ ] Content team deliverables are scheduled
