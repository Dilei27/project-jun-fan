# Product Vision — Project Jun Fan

> RC-01 Product Polish Review Board
> Como transformar o Project Jun Fan em um produto de referência.

---

## Visão 2026

O Project Jun Fan deve ser reconhecido como um **Product Operating System pessoal** — um ambiente onde engenheiros de QA, automação e IA podem apresentar seu trabalho não como portfólio, mas como **plataforma navegável**.

Não um currículo. Não um dashboard. Uma **narrativa viva de engenharia**.

---

## Diferenciais a Explorar

### 1. QA Command Center como Categoria
Nenhum outro portfólio de QA/automação existe como plataforma produto. O CC é um diferencial único. Deve ser tratado como **o hub que define a categoria**.

### 2. "Absorb. Refine. Build." como Método
A filosofia não é só um mantra — é um processo. Deve ficar visível na interface, não só no README.

### 3. AI Dock como Copiloto de Portfólio
Não um chatbot genérico. Um copiloto que conhece o conteúdo e guia o visitante.

### 4. ADRs como Conteúdo de Produto
Decisões técnicas são tratadas como parte do produto, não como documentação escondida.

### 5. Horizon Design System como Fundação
Design system próprio transmite maturidade de produto.

## O Que Deve Permanecer em Labs

| Ideia | Motivo |
|---|---|
| Voice commands | Tecnologia ainda não madura para o contexto; adicionaria complexidade sem ganho claro |
| PWA | Mobile-first ainda não é prioridade; vale testar como lab |
| Architecture Pipeline | Depende de mais produtos conectados; prematuro agora |
| AI real (serverless) | Requer backend seguro; custo operacional; manter mock até PRD v0.3 |
| Grafo de expertise interativo | Visualmente interessante, mas alto esforço para valor ainda não validado |
| Modo claro | Duplica manutenção de tokens; postergar até UI principal estar consolidada |

## Posicionamento Desejado

> "Project Jun Fan não é um portfólio. É a prova de que QA pode ser produto."

Em uma entrevista internacional, queremos que o recrutador diga:
- "Nunca vi um QA se apresentar assim."
- "Isso parece um produto de verdade, não um site pessoal."
- "O nível de documentação e design é incomum para QA."
- "Isso mostra pensamento de produto, não só código."
