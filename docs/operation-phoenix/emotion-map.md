# Emotion Map — Project Identity

> Qual sentimento queremos provocar em cada etapa da jornada.

---

## Mapa Emocional da Jornada

```
Entrada (Home / Link)
  │
  ├── Curiosidade ── "O que é isso?"
  │   Hero com identidade, nome diferente, mantra
  │
  ├── Interesse ── "Isso parece diferente"
  │   Status do ecossistema, 3 produtos como gateway
  │
  ├── Exploração (Command Center)
  │   │
  │   ├── Confiança ── "Isso foi bem construído"
  │   │   Métricas, timeline, architecture flow
  │   │
  │   ├── Profundidade (Product / Project pages)
  │   │   │
  │   │   ├── Admiração ── "Nunca vi QA assim"
  │   │   │   Problema → solução → impacto, decisões visíveis
  │   │   │
  │   │   └── Conexão ── "Quero saber mais"
  │   │       AI Dock, contato, docs
  │   │
  │   └── Docs
  │       │
  │       └── Utilidade ── "Isso realmente explica"
  │           Documentação clara, setup, arquitetura
  │
  └── Saída
      │
      └── Lembrança ── "QA Command Center"
          O conceito é memorável, a identidade fica
```

## Sentimentos por Componente

| Componente | Sentimento | Como |
|---|---|---|
| **Header** | Estabilidade | Fixo, backdrop-blur, sempre visível |
| **Hero (Home)** | Curiosidade | Nome único, mantra, badge de status |
| **Hero (CC)** | Impacto | Título grande, glow, badge "online" |
| **Status Strip** | Confiança | Produtos online, indicadores visuais |
| **Product Cards** | Interesse | Acento colorido, stack tags, hover |
| **Project Cards** | Credibilidade | Problema → solução → impacto |
| **Timeline** | Progressão | Evolução visível, ring no início |
| **Metrics** | Confiança | Números com contexto |
| **Decision Cards** | Transparência | Trade-offs visíveis, sem esconder |
| **Architecture Flow** | Clareza | Fluxo visual, engenharia visível |
| **AI Dock** | Utilidade | Respostas rápidas, links internos |
| **Command Palette** | Velocidade | Atalho, busca rápida |
| **Badge** | Confirmação | Status, tags, variantes semânticas |
| **Skeleton** | Paciência | Loading suave, sem surpresa |
| **404 / Error** | Transparência | Mensagem clara, ação de volta |
| **Footer** | Completude | Links, informação, encerramento |

## O Que NUNCA Queremos Provocar

| Sentimento | Onde Evitar |
|---|---|
| **Confusão** | Navegação ambígua, CTAs concorrentes |
| **Ansiedade** | Loading sem feedback, informação escondida |
| **Indiferença** | Design genérico, sem personalidade |
| **Desconfiança** | Links quebrados, métricas sem fonte |
| **Frustração** | Mobile quebrado, falta de busca |
| **Sobrecarga** | Excesso de cards, informação densa |
| **Tédio** | Páginas sem personalidade |

## A Emoção do Project Jun Fan em Uma Palavra

**"Precision"** — precisão técnica, precisão visual, precisão emocional.

Cada pixel, cada palavra, cada animação existe por um motivo. Nada é acidental. Nada é decorativo. O usuário sente que cada detalhe foi pensado — porque foi.
