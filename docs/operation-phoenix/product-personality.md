# Product Personality — Project Identity

> A personalidade do Project Jun Fan.

---

## Personalidade

**Arquiteto Silencioso**

- Introvertido, mas expressivo no trabalho
- Fala quando tem algo relevante a dizer
- Não busca atenção, mas quando aparece, entrega valor
- Perfeccionista funcional — cada detalhe tem propósito

**Referências de Personalidade:**

- **Linear** — a calma e precisão
- **Stripe** — a documentação como produto
- **Nothing** — o futurismo industrial
- **Rivian** — a tecnologia premium sem exibicionismo
- **Apple** — o refinamento silencioso

---

## Tom de Voz

| Contexto | Tom |
|---|---|
| Hero / Headings | Confiante, direto, sem hype |
| Descrições | Técnico, preciso, sem jargão |
| CTAs | Imperativos suaves: "Explorar", "Ver", "Abrir" |
| Erros | Informativos, nunca técnicos |
| AI Dock | Profissional, útil, sem personalidade forçada |
| Documentação | Tutorial, claro, exemplos práticos |
| Badges / Labels | Objetivos, sem adornos |

**O que NUNCA dizer:**
- "Revolucionário", "Disruptivo", "Transformador"
- "Inteligência artificial" (preferir "IA aplicada" ou "classificação")
- "Empoderando", "Mudando o mundo"
- Emojis na interface (exceto em demos)

## Emoção

Queremos que o visitante sinta, nesta ordem:

1. **Curiosidade** — "O que é isso?"
2. **Interesse** — "Isso é diferente"
3. **Confiança** — "Isso foi bem construído"
4. **Admiração** — "Nunca vi QA apresentado assim"
5. **Conexão** — "Quero saber mais / quero trabalhar com essa pessoa"

---

## Ritmo

- **Home**: apresentação calma, um elemento por vez
- **Command Center**: densidade média, navegação por seções
- **Produto**: leitura focada, informação hierarquizada
- **Docs**: referência, busca rápida
- **AI Dock**: conversacional, respostas curtas

---

## Elegância vs Minimalismo

**Elegância** — cada elemento tem propósito. Nada é decorativo.
**Minimalismo** — ausência de excesso. Mas não ausência de identidade.

O Jun Fan é **elegante antes de minimalista**. Bordas sutis, glows leves, motion discreto — mas todos com função.

---

## Tecnicidade

A interface não esconde que foi construída por um engenheiro. Ela celebra isso:
- Architecture flows visíveis
- Stack tags em cards
- Decisões técnicas como conteúdo
- Código bem formatado em docs
- Badges de status como indicadores de maturidade
