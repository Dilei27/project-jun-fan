# Brand Discovery — Project Identity

> Quem é o Project Jun Fan?

---

## Quem Somos

Project Jun Fan é o **Product Operating System** de um engenheiro de QA que evoluiu de testes manuais para automação, IA aplicada e arquitetura de sistemas.

Não somos um portfólio. Somos a prova de que QA pode ser tratado como disciplina de produto.

## Por Que Existimos

Projetos técnicos excelentes parecem pequenos quando apresentados como repositórios soltos, currículos ou dashboards genéricos. O Project Jun Fan existe para **elevar o padrão de como engenharia de qualidade é apresentada**.

Existimos para mostrar que:
- Automação não é só script — é arquitetura
- QA não é só teste — é engenharia de produto
- Portfólio não é só link — é plataforma

## Qual Problema Resolvemos

Projetos de QA, automação e IA aplicada são difíceis de apresentar:
- Repositórios soltos não contam a história
- Currículos não mostram arquitetura
- Dashboards genéricos não transmitem identidade

O Project Jun Fan resolve **como engenheiros de qualidade apresentam seu trabalho como produto**.

## Como Queremos Ser Percebidos

| Percepção | Como |
|---|---|
| **Premium** | Design escuro, tipografia limpa, espaçamento generoso |
| **Técnico** | ADRs, architecture flows, decisões visíveis |
| **Futurista** | AI Dock, motion funcional, terminal aesthetic |
| **Silencioso** | Sem exageros, sem neon, sem ruído visual |
| **Confiável** | Consistência, tokens, documentação como produto |

## O Que Nunca Devemos Parecer

| ❌ Evitar | Motivo |
|---|---|
| Dashboard genérico | Transmite monitoramento, não produto |
| Template SaaS | Falta personalidade |
| Portfólio Wix/WordPress | Amador |
| IA genérica (neon, roxo, circuitos) | Clichê |
| Currículo online | Subdimensiona o trabalho |
| Landing page de startup | Vazio de conteúdo |

## Valores Que Devem Aparecer Durante a Navegação

1. **Engenharia visível** — cada tela comunica que existe arquitetura por trás
2. **Simplicidade** — navegação clara, sem excesso de informação
3. **Reutilização** — padrões consistentes entre páginas
4. **Transparência** — decisões, trade-offs e impacto são públicos
5. **Evolução** — timeline mostra progressão, não estado final
6. **Identidade** — cada produto é único, mas pertence à mesma família
